Rotation 2 Dimensions in Shader :

The 2d rotation is a mathematical calculation of Matrix 2 by 2 represented in this way:

A B
C D

which makes two variable in the form of “vec2” for static displacement of the x/y axis by rotation axis “1.2,3.4” (1.2=x and 3.4=y) of “uv”.


“uv” is a mathematical calculation which allows you to redefine the correct center (to the nearest pixel) because the central pixel 0.0 is at the top left for a computer because it is the first pixel which loads

Here is the calculation:

vec2 uv=(gl_FragCoord.xy/v2Resolution-0.5)/vec2(v2Resolution.y/v2Resolution.x,1);

This explanation is only theoretical but the code in all this?

And here it is:

//Le Calcule Matriciel mat2
mat2 R2d(float r){return mat2(cos(r),sin(r),-sin(r),cos(r));}

void mainImage(out vec4 fragColor,in vec2 fragCoord){

//Le Calcule uv pour centrer le pixel 0,0
vec2 uv=(fragCoord/_res-0.5)/vec2(_res.y/_res.x,1);

//Le nom incrit vec2 "R2d" est le nom utilisable
vec2 R2d = uv * R2d(_t/4.);

//Le vec4 de sortie de l'image
fragColor=vec4(0.,0.,0.);
}
